# BB Toolkit Adapters

## Why adapters exist

The BB Toolkit has a single canonical configuration in `CLAUDE.md` and `.claude/skills/`. Adapters make the same skill catalog, dispatch rules, and safety rules available to other agent environments (OpenCode, Codex, Cursor, etc.) without duplicating logic.

When you add a new skill or update a dispatch rule, run the adapter and all agent platforms get the update.

## How to preview changes

```bash
python3 tools/adapters/render_adapter.py preview --target opencode
python3 tools/adapters/render_adapter.py preview --target cursor
python3 tools/adapters/render_adapter.py preview --target codex
python3 tools/adapters/render_adapter.py preview --target generic
```

This shows a unified diff of what would change, without modifying any files.

## How to apply safely

When the preview looks correct:

```bash
python3 tools/adapters/render_adapter.py apply --target opencode
```

The apply command:
- Creates a `.bak` backup of every file it overwrites
- Wraps generated content in adapter ownership markers
- Records the generation in `tools/adapters/state.yaml`
- Never exports `.bb/`, `output/`, or `engagements/` contents

To apply all targets at once:

```bash
for target in claude-code opencode codex cursor generic; do
  python3 tools/adapters/render_adapter.py apply --target "$target"
done
```

## How to uninstall/remove generated content

```bash
# Remove all generated files for a specific target
python3 tools/adapters/render_adapter.py uninstall --target cursor

# Remove all tracked files across all targets
python3 tools/adapters/render_adapter.py uninstall

# Skip confirmation prompt
python3 tools/adapters/render_adapter.py uninstall --target cursor --force
```

Uninstall preserves user-written content by only removing adapter-generated sections (marked with `<!-- BB-ADAPTER-GENERATED -->` comments). If a file contains both adapter and user content, it is skipped.

## Platform-specific notes

### Claude Code
- Target: `claude-code`
- Output: `CLAUDE.md` (in repo root)
- Sections: header, skill_catalog, dispatch_rules, safety_rules, environment
- Generated sections are wrapped in ownership markers

### OpenCode
- Target: `opencode`
- Output: `AGENTS.md` (in repo root)
- Full file is adapter-generated

### Codex
- Target: `codex`
- Output: `.codex/instructions.md`
- The `.codex/` directory is created if it doesn't exist

### Cursor
- Target: `cursor`
- Output: `.cursor/rules/bb_toolkit.md`
- The `.cursor/rules/` directory is created if it doesn't exist

### Generic
- Target: `generic`
- Output: `docs/bb_toolkit_agent_guide.md`
- Useful as a reference or for custom integrations

## Ownership tracking

The file `tools/adapters/state.yaml` tracks:
- Which files were generated by the adapter
- When they were generated
- Section-level hashes (to detect manual edits)

If a section hash changes from the recorded value, the adapter warns that a user has manually edited a generated section, and will not overwrite it without confirmation.

## Safety guarantees

- The adapter never includes auth-related variables in templates
- Files in `.bb/`, `output/`, `engagements/`, and `evidence/` are never exported
- All writes are diff-first (preview before apply)
- Original files are always backed up to `.bak` before overwrite
